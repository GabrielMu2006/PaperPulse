# PaperPulse UI 设计规格

更新日期：2026-07-14  
定位：当前 macOS 实现的页面、布局、视觉与交互基准；同时是 Windows 原生客户端收敛 UI 的设计输入。

## 1. 使用方式与边界

本文件描述的是已经落地的 macOS UI，不是营销概念稿。实现依据是：

- `Apps/PaperPulseMac/Sources/PaperPulseMac/MacRootView.swift`
- `Apps/PaperPulseMac/Sources/PaperPulseMac/MacLibraryView.swift`
- `Apps/PaperPulseMac/Sources/PaperPulseMac/MacFeedEditorView.swift`
- `Apps/PaperPulseMac/Sources/PaperPulseMac/MacSettingsView.swift`
- `Apps/PaperPulseMac/Sources/PaperPulseMac/MacFullInterpretationView.swift`

Windows 端不需要像素级复制 SwiftUI，但应把本文件的**信息优先级、阅读密度、色彩角色、控件语义和交互状态**实现为符合 WinUI 3 的原生界面。当前 `Apps/PaperPulseWindows/src/PaperPulse.Windows/MainWindow.xaml` 是功能验证用的三栏壳，不是最终视觉规范。

本规格只约束表现层。不得因 UI 实现而修改 iOS、`PaperCore` 检索/下载/LLM 流程、SwiftData schema 或 Keychain/PasswordVault 的存储边界。

## 2. 设计意图

PaperPulse 是一个专业科研论文阅读与管理工具，而不是资讯流或营销产品。界面应同时满足两件事：

1. **可靠、可扫描。** 用户可以在一眼内辨认订阅、论文归属、当前选择、PDF 是否存在、解读是否可用，以及一项任务是否正在进行。
2. **有品牌活力，但不喧宾夺主。** 深色学术工作台承载红、洋红、紫色脉动；暖白“纸张”表面只用于需要长时间阅读的论文内容。

全窗使用固定暗色品牌外壳，不跟随系统浅色模式。颜色用于建立层级和状态，不用于制造大量装饰性卡片或让每一个普通控件都变成渐变按钮。

## 3. 视觉 Token

### 3.1 色彩角色

| Token | 近似色值 | 角色与使用位置 |
| --- | --- | --- |
| `midnight` | `#05030F` | 最深背景、窗口底色、阴影基础。 |
| `deepBlue` | `#080D29` | 外壳蓝黑过渡。 |
| `deepPurple` | `#290538` | 外壳紫黑过渡。 |
| `pulseRed` | `#F01A1F` | 主要动作、警示性强调、解读进度。 |
| `pulseMagenta` | `#EB1FA3` | 活跃分组、论文图标、渐变中段。 |
| `pulsePurple` | `#6B2BEB` | 渐变末端与背景深处。 |
| `warmGold` | `#FFA11F` | 收藏星标与少量正向强调。 |
| `paper` | `#FFF7E8` | 阅读内容卡片的暖白纸张面。 |
| `paperSoft` | `#FFEEDA` | 纸张渐变末端。 |
| `paperInk` | `#1F1A21` | 纸张面主文字。 |
| `paperSecondary` | `#5C4F5C` | 纸张面辅助文字与 metadata。 |
| `glassFill` | 白色 7.5% 透明 | 深色壳上的分区面。 |
| `glassStroke` | 白色 16% 透明 | 深色壳上的 1 px 边界。 |

### 3.2 渐变与背景

- **主强调渐变**：左上红色，经过洋红，落到右下紫色。用于主按钮、活跃计数、品牌标记与少量焦点状态。
- **外壳背景**：`midnight -> deepBlue -> deepPurple -> 暗红紫 -> 深蓝紫` 的斜向层次；左上有偏红的柔光，右下有偏紫的柔光，底部有低对比度的洋红/紫色流光带。
- **阅读面**：`paper -> white -> paperSoft` 的轻微渐变。它必须优先保证黑色正文可读性，不能叠加高饱和渐变。

### 3.3 圆角、边框与阴影

- 深色玻璃分区、表单分区、阅读卡片：`10 pt` 连续圆角。
- 紧凑选择态与输入框：`8 pt`；较小的品牌标记：`6-9 pt`。
- 深色玻璃分区使用 `1 px` 半透明白边；阅读卡片使用 `1 px` 较亮白边与低强度暗色投影。
- 页面不使用“卡片套卡片”。侧边栏中的“订阅”和“论文库”是顶层分区；阅读区的卡片用于真正独立的论文信息块或章节，不再额外包一层浮动卡片。

### 3.4 字体与密度

使用系统字体，不引入品牌字体。字级遵从 macOS 可读性而非海报式展示：

| 层级 | 现有实现 | 用途 |
| --- | --- | --- |
| 页面标题 | `title2.bold` | 设置、新建/编辑订阅、论文标题。 |
| 卡片/区块标题 | `headline` | 简介、表单分区、完整解读章节。 |
| 列表主行 | `callout.semibold` | 订阅名、论文题目。 |
| 正文 | `body` 或 `17 pt` | 摘要和完整解读；解读正文 `lineSpacing(4)`。 |
| 元数据 | `caption` / `caption2` | 作者、日期、计数、状态、辅助说明。 |

- 侧边栏优先保证扫描密度：论文标题最多两行，作者一行，短简介最多两行。
- 论文详情标题最多四行，作者最多两行；正文允许完整展开与文本选择。
- 所有中英文文案可换行；按钮和表格标签不得因长中文或英文模型名发生裁切。

## 4. 全局窗口结构

### 4.1 主窗口

```text
┌─────────────────────────────────────────────────────────────────┐
│ 深色品牌外壳                                                     │
│ ┌──────────── Sidebar ───────────┐ ┌──── 论文详情 / 阅读区 ───┐ │
│ │ 品牌标头 + 当前状态             │ │ 左：论文信息或完整解读    │ │
│ │ 搜索                             │ │ 可拖动分隔条              │ │
│ │ 全部论文 / 收藏 分段切换         │ │ 右：本地 PDF              │ │
│ │ 设置 | 新建订阅                  │ └──────────────────────────┘ │
│ │ 订阅分区                         │                              │
│ │ 论文库分区（按订阅折叠）         │                              │
│ └─────────────────────────────────┘                              │
└─────────────────────────────────────────────────────────────────┘
```

- 使用 `NavigationSplitView`，最小窗口为 `900 x 600`。
- Sidebar 宽度：最小 `300 pt`、理想 `340 pt`、最大 `420 pt`。
- Sidebar 与详情区共用整窗深色背景；不可出现默认白色或灰色 source list 底板。
- 未选择论文时，详情区居中显示单一玻璃空状态：“未选择论文 / 从论文库中选择一篇论文开始阅读”。

### 4.2 详情阅读分栏

- 详情区使用左右可拖动分栏。默认比例 `1:1`（`0.5`），可在 `0.25 ... 0.75` 之间调整。
- 分隔条宽 `8 pt`，有低对比度轨道和 `2 pt x 48 pt` 抓手。用户调整后的比例持久化。
- 左侧为论文信息或完整解读；右侧固定为本地 PDF。无 PDF 时右侧显示玻璃空状态，而不是留白。
- 打开完整解读时，主 Sidebar 收起，保持“完整解读 + PDF”占据整个工作区，阅读密度不被导航打断。

## 5. Sidebar 规格

### 5.1 顶部顺序与间距

顶部内容水平内边距 `14 pt`，上 `12 pt`、下 `12 pt`，依次为：

1. **品牌标头**：`34 x 34 pt` 红-洋红-紫渐变方形标记，内有书本与脉冲图标；右侧为 `PaperPulse` 与一行状态。
2. **搜索框**：左侧放大镜，右侧清除按钮；`8 pt` 圆角，深色半透明底，支持搜索标题、作者、摘要。
3. **范围分段控制**：`全部论文 / 收藏` 两项；选中项为主强调渐变，未选中项为半透明深色。
4. **并列次要动作**：`设置` 与 `新建订阅`。均为带图标的低强调按钮，不应与纸飞机推送争夺注意力。

下方是可滚动内容，左右内边距 `12 pt`，分区之间 `14 pt`。

### 5.2 订阅分区

- 使用一个玻璃分区，默认内边距 `12 pt`，标题为全大写/紧凑的“订阅 / FEEDS”。
- 订阅行包含：`22 x 22 pt` 脉冲标记、订阅名、分类或关键词摘要，以及右侧独立的纸飞机圆形按钮。
- 选中订阅时，标记改为主强调渐变；订阅名称提高字重。纸飞机永远对应**该行订阅**。
- 运行中纸飞机替换为小型进度圈并禁用，不隐藏该位置，防止行宽跳动。
- 行右键菜单提供编辑；允许删除时才出现危险操作。

### 5.3 论文库分区

- 同样使用一个玻璃分区，标题右侧显示当前筛选结果总数的小胶囊计数。
- 每个订阅以及“未归类”都是可折叠分组。分组标题包含文件夹图标、名称和数量；活跃订阅使用洋红图标与渐变数量胶囊。
- 点击订阅行时必须：选中订阅、展开对应论文组、折叠其他订阅与未归类。用户仍可以手动开合任意组。
- 论文列表项包含：文档图标、标题、收藏星标、作者、短简介与日期。标题最多两行，作者一行，短简介两行。
- 选中论文使用浅色透明底与左侧 `3 pt` 渐变强调条；没有选中态时不得用大面积高饱和填色。
- 空分组显示一行“尚未推送论文”，保持低强调，不出现大插画或占满区域的空卡。

## 6. 论文详情与 PDF 工作区

### 6.1 左侧论文信息

左侧为可滚动阅读列，外边距 `20 pt`，信息块间距 `14 pt`。

1. **论文身份卡**：暖白阅读面，默认内边距 `18 pt`。包含标题、作者、metadata pills 与收藏按钮。
   - metadata 依次可包含：来源、期刊/会议、发布日期、引用数。
   - metadata pill 使用浅红半透明底、`paperSecondary` 文字；是事实信息，不应显得像可点击主按钮。
   - 收藏按钮位于右上；收藏后使用金色星标。它是唯一保留的个人分类入口。
2. **简介卡**：暖白阅读面。标题为“简介 / Brief”，右上放完整解读主操作。
   - 没有完整解读：显示“生成完整解读”。
   - 已有完整解读：显示“打开完整解读”。
   - 失败后按钮改为“重新生成完整解读”，下方展示简短红色错误文本。
3. **来源卡**：只有存在来源 URL 时出现；是低强调的“打开来源页面”链接操作。

主操作使用原生 prominent button 语义并染为 `pulseRed`，不将普通 source link 或收藏做成相同等级的渐变大按钮。

### 6.2 PDF 区

- 有本地 PDF：使用原生 PDF 阅读组件全高显示，背景保持深色。
- 无 PDF：居中玻璃空状态，说明“尚未下载 PDF / 仅公开可访问的 PDF 会保存到本地”。
- PDF 是与论文信息同等重要的工作面；不能被缩成右下预览卡或只用“下载”链接替代。

### 6.3 生成完整解读状态

生成期间，简介卡原有正文位置改为显式状态区：

- 常规尺寸进度圈，色调为 `pulseRed`。
- 标题：“正在生成完整解读”。
- 辅助文字：“正在读取 PDF 并整理基于原文的解读；你可以继续浏览其他论文。”
- 该状态按论文 ID 关联；用户切换到其他论文时可继续操作，不能把整个窗口锁死。
- 没有 API Key 时，必须明确提示用户去“设置”保存 API 配置；**不可**以本地短简介或规则摘要假冒完整解读。

## 7. 完整解读工作区

- 进入后收起 Sidebar；左侧显示 `MacInterpretationPane`，右侧仍显示相同论文的 PDF。
- 顶部身份卡显示：完整解读标题、论文标题、模型、生成时间、文本来源范围和本地 Markdown 文件名（可用时）。
- 右上保留两个明确动作：`删除` 与 `关闭`。删除为 destructive，必须出现确认弹窗。
- 每个解读章节是独立暖白阅读卡，章节标题左对齐、页码 anchors 右对齐并使用 `pulseRed`。
- 解读正文为 `17 pt`，行距 `4 pt`，可文本选择；不强行压缩成摘要列表。
- 删除只影响当前论文的完整解读 metadata 与 Markdown，完成后返回论文信息 + PDF 状态。

## 8. 新建/编辑订阅 Sheet

### 8.1 容器

- 固定尺寸 `720 x 700 pt`，深色品牌外壳。
- 上部内容可滚动，四周 `24 pt`；底部 footer 固定，内边距 `16 pt`，顶部有 `1 px` 半透明分割线。
- 页头使用 `38 x 38 pt` 渐变图标、`title2.bold` 标题、两行以内副标题。

### 8.2 表单顺序

每个区块都是 `10 pt` 圆角的半透明深色表单面，默认内边距 `18 pt`，内部纵向间距 `12 pt`：

1. **订阅**：名称、arXiv 分类。
2. **关键词（任一匹配）**：关键词库自适应多列复选框 + 自定义关键词文本框。
3. **筛选要求**：排除关键词、机构、期刊或会议；下方必须保留“留空代表任意，但仍执行权威性判断”的说明。
4. **学术来源**：arXiv、OpenAlex、Crossref 复选框。
5. **筛选**：每次篇数、检索范围天数的 stepper。

footer 左侧为取消，右侧为红色 prominent 的保存订阅。名称、至少一类关键词/分类、至少一个来源未满足时禁用保存。

## 9. 设置窗口

- 设置窗口宽 `620 pt`、最小高 `680 pt`，深色外壳 + 可滚动内容 + `22 pt` 外边距。
- 顶部使用与订阅编辑器一致的页面标头：`38 x 38 pt` 齿轮渐变图标、标题“设置”、副标题。
- 分区顺序固定：`语言`、`关键词库`、`模型配置`、状态（仅有状态时）、`存储`。
- `界面语言` 与 `简介语言` 是两个独立 Picker，不能相互修改或共用一个默认值。
- 模型配置在一个表单分区中按“当前配置 -> 新建/删除 -> 服务商 -> API 格式 -> Base URL -> Model -> API Key -> 保存/测试”排列；配置说明位于底部。
- API Key 输入使用 secure field；配置说明应强调 Key 只保存在本机 Keychain。测试 API 是强调动作，保存配置为次级动作。
- 存储区只包含危险操作“清除未归类文章”；确认弹窗明确会删除未归类论文、简介与本地文件。

## 10. 状态、可访问性与本地化

- 每一个异步任务都需要稳定位置的进度或错误状态，不能仅依赖控制台/Toast。
- 不直接向用户显示内部 HTTP 错误码、Provider 原始 exception 或密钥信息；改为来源不可用、可重试、配置缺失等可行动文案。
- 图标按钮必须带 tooltip / accessible name：纸飞机、删除、关闭、搜索清除等尤其如此。
- 中文与英文切换影响所有 UI 文案；模型名、来源名、API 协议名可保留原文。
- 英文与中文长度不同：侧边栏、分组标签、按钮和 metadata 必须允许文本换行/截断策略，而非让控件改变整体布局。

## 11. Windows 映射规则

Windows 采用 C# + WinUI 3，不复用 SwiftUI 代码；但实现者应把下面的视觉结构映射到 Fluent/WinUI 控件：

| macOS 概念 | Windows 建议 | 不可丢失的结果 |
| --- | --- | --- |
| Sidebar | 左侧固定导航栏/TreeView + ScrollViewer | 顶部品牌、搜索、范围切换、订阅、分组论文库的顺序。 |
| DisclosureGroup | 可展开的 TreeView group 或 Expander | 订阅和未归类均可折叠，选择订阅时按规则收拢其他组。 |
| `MacBalancedSplitView` | `Grid` + `GridSplitter` | 默认 1:1、范围限制、持久化比例、解读与 PDF 并排。 |
| `MacSurfaceCard` | 自定义 WinUI ContentControl/Border | 暖白可阅读表面、深色正文、非营销式阴影。 |
| `MacGlassPanel` / `MacFormCard` | 深色 translucent Border | 保持外壳内的可扫描分区，不做默认灰色面板。 |
| Paper plane | 只带图标、hover tooltip 的紧凑 Button | 每一订阅都有独立手动推送入口与进度占位。 |
| PDFView | WebView2 本地 PDF | PDF 是并排工作面，不是附件链接。 |

Windows 可以使用符合系统规范的窗口控件、键盘焦点与 hover 样式；不能因此删掉深色品牌外壳、暖白阅读面、明确的 action hierarchy 或上述信息顺序。

## 12. 新聊天执行约束

另一个聊天中的模型在修改 UI 前，应先将本文件、`docs/macos/ui-handoff.md` 与 `docs/development/windows-migration-handoff.md` 作为约束读取。实现顺序应为：

1. 先复述理解到的页面结构、token 和不可变交互契约。
2. 给出只涉及表现层的文件清单与阶段计划。
3. 每一阶段做完运行该平台可用的构建/测试，且不触碰业务/存储边界。
4. 在 Windows 分支上，macOS 只能验证可移植 C# 模块；WinUI/MSIX 以 Windows CI 与 Windows 机器验证为准。
